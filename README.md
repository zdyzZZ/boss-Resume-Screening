# 简历筛选系统

## 目录结构

```
resume_filter/
├── app.py                # 主程序(数据库/解析/Web 全部在内)
├── init_db.sql           # 建表脚本,首次运行前先执行
├── requirements.txt      # 依赖清单
├── start.bat             # Windows 启动批处理
├── templates/
│   └── index.html        # 前端页面
└── README.md
```

## 前置要求

1. Python 3.11(推荐)
2. MySQL 已就绪,连接信息见 `app.py` 里的 `Config` 类
3. 简历按日期存放:`D:\boss直聘\2026-04-23\xxx.pdf`

## 首次部署

1. 在 MySQL 执行建表脚本:
   ```
   mysql -h 192.168.0.196 -u root -p cx < init_db.sql
   ```
   (或者用 Navicat 之类的工具打开 init_db.sql 执行)

2. 安装依赖:
   ```
   pip install -r requirements.txt
   ```

3. 启动:
   ```
   python app.py
   ```
   或双击 `start.bat`

4. 浏览器访问:
   - 本机:http://localhost:5000
   - 局域网其他机器:http://本机IP:5000

## 任务计划程序(可选)

如果希望开机自动启动,在 Windows 任务计划程序中:
- 触发器:计算机启动时
- 操作:启动程序,程序选 `start.bat` 的完整路径
- 勾选"不管用户是否登录都要运行"

## 功能说明

- **默认看今天**:页面打开后自动加载今天日期的简历文件夹
- **切换日期**:用页面顶部的日期选择器切换,会自动加载该日期已解析的数据
- **重新解析**:点"重新解析"按钮,强制扫描当前日期的文件夹,新增的 PDF 会被解析入库
- **只处理 PDF**:其他格式文件会被跳过
- **重复检测**:通过文件 MD5 判断,内容没变就不重新解析
- **编辑和操作共享**:所有人看到相同的手动编辑结果和通过/否决状态
- **筛选独立**:每个人的筛选条件是浏览器本地的,互不影响

## 常见问题

**Q: 文件夹不存在怎么办?**
A: 后端会返回空列表,页面显示"暂无简历"。把文件放进对应日期目录后,刷新即可。

**Q: 修改了简历 PDF 怎么重新解析?**
A: 文件内容变化(MD5 变了)会自动重新解析。或者点"重新解析"按钮强制重跑。

**Q: 数据库连错信息?**
A: 改 `app.py` 顶部的 `Config` 类里的 DB_* 参数,重启服务。
